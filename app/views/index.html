Changes

This version is the initial design prototype of the case list, for a single court.

We've added anonymised / dummy Libra case list data for two days in advance. For each case users can view name, probation record, offences, case number, session (am/pm), courtroom.

The case list includes a filter component to allow defining a subset of cases. The user can segment cases by courtroom, session (am/pm), probation record.

We've added a flow for adding overnight / remand cases. This includes a screen for matching defendant details against possible records found in nDelius. One of the records on the 'Confirm offender record' (matching) screen links to an offender summary screen (dummy nDelius data).
